# Overall Vibe

## Formality
I view something like the ZTF homepage (https://irsa.ipac.caltech.edu/Missions/ztf.html) to be very formal, in the sense that it feels like the design was following a template that was created by a funding agency. I want to contrast that with the formality of something like the home page of the AAVSO (https://www.aavso.org/), which I consider to be a much better balance of formal/leisure ratio.
I think what I mean when I say leisurely is really about whether it feels like someone with frontend/design experience was a part of the process.
I want this to feel closer to the leisure side, although not quite as leisurely as the AAVSO page.


## Busyness
I want this to be/feel very light weight and very straightforward. Not quite to an extent that I would call it minimalist, but close to it.


## Formatting
For the homepage, I want things to look nice and be easily navigable. That is top priority. For the individual nova pages, I’d like the formatting to be such that folks can easily grab/copy/download information from it (if possible). Like, I’ve frequently run into issues where, e.g., tables are hard to copy/paste into something new. If possible, I’d like to avoid that.


## Home page layout
I like what we’ve been talking about with these hybrid pages. Have a sample of the catalog and maybe a link to it, but also have other stuff on it. I’d like to at least try this. If it doesn’t work out, I’d like our fall back to be a catalog home page.



# Other notes:
- I’d like to contrast some websites that have things that I don’t like: I love and appreciate Vizier (https://vizier.cds.unistra.fr/viz-bin/VizieR) and Simbad (https://simbad.cds.unistra.fr/simbad/), and I would love to have my usability be anywhere close to these pages. But their UI/UX is so so so busy. I think that I would like to have some features similar to the scrolling tables in Vizier for an advanced search option (so, e.g., folks can quickly find novae with specific data traits). But yeah, I would like to have a much simpler/cleaner interface than these two.
- For the actual catalog, I want something that has the same general layout as the Open Exoplanet Catalog (https://www.openexoplanetcatalogue.com/systems/), but has some CSS flair to it, so it doesn’t just feel like the defaults were used, and also incorporates/embeds some visualizations into it (maybe just one, like the light curve?).
- I want there to be a way for folks to quickly and easily see data right out of the gate, so they can get a sense of what this catalog contains. So, e.g., they can click on one of the most popular novae and see it’s nova page right away.
